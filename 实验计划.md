## 一、项目背景

复现并改进论文，在 ESConv 数据集（8种情感支持策略）上研究如何用 GRPO 强化学习修正策略偏见。核心问题：SFT 只学会了最高频的 Question 策略（ci=0.85），其余7种策略 ci=0.00，模型完全无法选择。

**硬件**：4× A100 80GB，CUDA 12.6
**Base Model**：`/mnt/teamdrive/models/Qwen2.5-7B-Instruct`（本地已有，无需下载）
**conda 环境**：`verl`（已安装 trl 0.29.1, deepspeed 0.18.8, transformers 5.3.0）

## 三、核心发现

### 知识边界分布（v2）
```
HK（高确信，ci=1.0）:  16.44%  → Question 策略主导
WK（弱确信，0<ci<1）:   2.50%
UK（未知区，ci=0）:     81.06%  → 其余7种策略全在此
```

### 各策略学习情况
| 策略 | v2 ci | Agent 准确率 | 说明 |
|------|-------|------------|------|
| Question | 0.852 | 5.2% | SFT强，Agent弱（标注风格差异） |
| Reflection of feelings | 0.000 | 23.4% | SFT完全不会，Agent 有效 |
| Providing Suggestions | 0.000 | 40.7% | Agent 最强 |
| Affirmation & Reassurance | 0.000 | 29.9% | |
| Others | 0.000 | 40.4% | 语义混杂垃圾桶类别 |
| Restatement or Paraphrasing | 0.000 | 9.9% | |
| Information | 0.000 | 4.7% | |
| Self-disclosure | 0.000 | 2.5% | |

**核心问题**：81% UK 样本导致标准 GRPO 奖励全零，只会强化 Question 偏见。

## 四、实验计划

### Exp 1：Standard GRPO（基线）
```bash
conda activate verl
cd /home/aiscuser/projects/RLforESC

accelerate launch --num_processes 4 scripts/train_grpo.py \
    --mode standard \
    --model_path checkpoints/qwen25-esc-v2 \
    --output_dir checkpoints/grpo-standard \
    --num_train_epochs 1 --num_generations 8 --beta 0.001
```
**预期**：Question ci 继续升高，其余策略仍 0，证明基线失效

### Exp 2：Dual-reward GRPO（论文复现，核心）
```bash
accelerate launch --num_processes 4 scripts/train_grpo.py \
    --mode dual \
    --model_path checkpoints/qwen25-esc-v2 \
    --boundaries_path data/knowledge_boundaries_v2/train_turns_with_boundaries.json \
    --output_dir checkpoints/grpo-dual \
    --num_train_epochs 1 --num_generations 8 --beta 0.001
```
**预期**：UK 样本通过熵奖励被引导，部分非 Question 策略 ci 从 0 提升

### Exp 3（改进）：课程学习 GRPO
- Phase 1：只用 HK+WK 样本（19%）训练 300 steps
- Phase 2：引入全量 UK 样本继续训练
- 需修改 `scripts/train_grpo.py`：`load_dual_dataset` 加 `filter_region` 参数

### Exp 4（改进）：Others 策略细化
- 将 Others（17.5%，ci=0，语义混杂）拆分为 Opening/Acknowledgment/Closing
- 新增 `scripts/refine_others_strategy.py`
- 修改数据清洗流程，重新训练

### Exp 5（改进）：评估升级
- 现有：strategy accuracy（受标注主观性限制，上限~40%）
- 引入：BLEU/ROUGE、Empathy Score、LLM-Judge（GPT-4/Claude）

## 五、评估方式

每轮 GRPO 后运行：
```bash
# 1. 重新做知识边界分析
python scripts/delineate_knowledge_boundary.py \
    --model_path checkpoints/grpo-dual \
    --output_path data/knowledge_boundaries_grpo/

# 2. 分析边界分布变化
python scripts/analyze_knowledge_boundary.py \
    --input data/knowledge_boundaries_grpo/train_turns_with_boundaries.json \
    --output results/boundary_analysis_grpo/

# 3. 测试集策略覆盖
python scripts/test_model.py --model_path checkpoints/grpo-dual --mode compare
```
