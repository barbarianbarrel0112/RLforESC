## 一、研究背景与目标

本研究聚焦于情感支持对话（Emotional Support Conversation, ESC）任务。该任务的核心挑战在于：模型不仅需要生成语言流畅的回应，还需要在对话的不同阶段选择恰当的支持策略——例如先共情、后建议，而非开口就给方案。

ESConv 数据集定义了八种支持策略：**提问（Question）、重述（Restatement or Paraphrasing）、情感反射（Reflection of feelings）、自我披露（Self-disclosure）、肯定与安抚（Affirmation and Reassurance）、提供建议（Providing Suggestions）、提供信息（Information）、其他（Others）**。

研究目标是让模型能够根据对话上下文，准确选择并执行合适的策略，最终提升情感支持的质量与合理性。

## 四、知识边界划分（v1）

### 4.1 方法说明

为了定量分析 SFT 模型在各策略上的学习程度，我们引入了**知识边界划分**（Knowledge Boundary Delineation）方法。

对训练集中每一个助手回合，用 SFT 模型在温度 0.4 下采样 K=8 次，统计生成结果中策略标签命中率：

$$c_i = \frac{\text{命中次数}}{K}$$

根据 $c_i$ 将每个回合划分为三类：

| 类别 | 条件 | 含义 |
|------|------|------|
| **HK**（高确信） | $c_i = 1.0$ | 模型每次都能选对策略 |
| **WK**（弱确信） | $0 < c_i < 1$ | 模型有时能选对 |
| **UK**（未知区） | $c_i = 0$ | 模型从未选对 |

二值熵 $e_i = -c_i \log_2 c_i - (1-c_i)\log_2(1-c_i)$ 衡量模型在该回合上的不确定程度。

### 4.2 v1 整体结果

```
区域分布（训练集，13,952 回合）：

 HK  ██████                    1,802  (12.93%)
 WK  ███                         711   (5.10%)
 UK  ████████████████████████  11,439  (81.97%)
```

**超过 82% 的训练样本处于未知区**，意味着 SFT v1 模型对绝大多数回合的策略完全没有把握。

### 4.3 v1 各策略知识分布

## 五、策略分布深度分析

### 5.1 Others 策略专项分析

Others 策略占训练集 17.5%（2,435 条），是第二大策略，但 ci=0.000，模型完全无法学习。

对 Others 策略的 40 条随机样本进行人工标注分析，发现其内容实质上覆盖四类截然不同的对话行为：
### Others 策略
| 子类型 | 比例 | 示例 |
|--------|------|------|
| 问候/开场 | ~15% | "Hi, what's on your mind today?" |
| 告别/收尾 | ~35% | "Take care of yourself, I'm always here." |
| 简短确认 | ~25% | "I see. Go on." / "That sounds really hard." |
| 话题过渡 | ~25% | "Let's talk a bit more about that..." |

**Others 的本质是一个语义混杂的垃圾桶类别**，不同子类型之间没有共同的语言特征，模型无法学会何时该用它。

### 5.2 策略转移矩阵（训练集）

分析相邻回合之间的策略转移，最高频的自环是：

- Others → Others：1,754 次（最高）
- Question → Question：1,023 次
- Providing Suggestions → Affirmation：712 次

Others 的高度自环印证了其"对话维持"功能——连续的问候、确认和过渡语形成一串 Others 标注。

## 八、实验三：基于提示工程的 Agent 方案

### 8.1 设计动机

SFT 方案存在两个结构性问题：

1. **策略分布偏斜**：Question 在训练集中频率最高，SFT 形成极强的 Question 偏见；
2. **监督信号不足**：81% 的样本处于 UK 区，若直接进行 GRPO 强化学习，这些样本奖励为 0，梯度接近零，无法有效训练。

因此转向 **Agent 方案**：不对模型进行策略级的监督微调，而是通过精心设计的系统提示，引导模型利用其预训练阶段积累的情感理解知识进行策略推理。

### 8.2 Agent 架构

```
┌─────────────────────────────────────────────────────┐
│                   System Prompt                     │
│  ┌────────────────────────────────────────────────┐ │
│  │  角色设定：你是一名专业情感支持顾问             │ │
│  │  输出格式：<thinking>/<strategy>/<response>    │ │
│  │  8种策略定义（基于 ESC 理论）                  │ │
│  │    + 每种策略 2 条 few-shot 示例（来自数据集）  │ │
│  │  注意事项：先情绪，后建议                       │ │
│  └────────────────────────────────────────────────┘ │
│                   User Message                      │
│  ┌────────────────────────────────────────────────┐ │
│  │  [Conversation so far]                         │ │
│  │  User: ...                                     │ │
│  │  Counselor: ...                                │ │
│  │  [Current user message]                        │ │
│  │  User: ...                                     │ │
│  └────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
             ↓  Qwen2.5-7B-Instruct 推理
┌─────────────────────────────────────────────────────┐
│  <thinking>用户当前情绪是...，对话处于...阶段，     │
│  最适合的策略是...因为...</thinking>               │
│  <strategy>Affirmation and Reassurance</strategy>   │
│  <response>你已经努力了很久...</response>           │
└─────────────────────────────────────────────────────┘
```

**链式思考（CoT）** 让模型先分析用户情绪状态和对话阶段，再作出策略选择，而不是直接输出答案。这使策略选择过程可解释、可审计。

### 8.3 few-shot 示例选取方法

从训练集中为每种策略各选 2 条代表性示例，筛选标准：

- 对话综合评分 ≥ 4.0（用户和支持者双方均评价高）
- 回应内容充实（非 Others 类：≥ 25 字符；Others 类：≤ 150 字符）
- 上下文不超过 6 轮（避免系统提示过长）
- 从最高质量的 20 条候选中随机采样（保证多样性）

实际共选取 16 条示例（每策略 2 条）。

### 8.4 评估方法与规模

- 评估集：测试集全量，128 对话，1,683 个助手回合
- 评估方式：逐轮模拟，用前 i 轮作为历史，预测第 i+1 个助手回合的策略
- 并行方式：4 卡数据并行，每卡处理 32 个对话，总用时约 32 分钟
- 生成参数：temperature=0（贪婪解码），max_new_tokens=384

**整体策略准确率：23.6%（377/1,596 回合）**

### 8.8 Question 策略为何表现差

Question 准确率仅 5.2%，混淆矩阵揭示了原因：

```
数据集标注为 Question 的 289 个回合，模型预测为：

→ Reflection of feelings       78 次 (27.0%)  ← 模型认为应先共情
→ Others                       68 次 (23.5%)  ← 模型认为是过渡性回应
→ Affirmation and Reassurance  65 次 (22.5%)  ← 模型认为应先安抚
→ Providing Suggestions        36 次 (12.5%)
→ Restatement or Paraphrasing  26 次  (9.0%)
```

**模型行为与数据集标注风格的分歧**：

## 十、主要发现与结论

**1. SFT 存在结构性瓶颈，与超参数无关**

无论是 v1 还是 v2，SFT 都只能让模型学会最频繁的 Question 策略，其余 7 种策略 ci 全部为零。根本原因是 SFT 的监督信号是"在这条数据上输出这个标签"，无法教会模型"在什么情况下该用什么策略"的全局决策能力。

**2. 知识边界分布极度不均，直接影响 GRPO 的可行性**

81% 的训练样本处于 UK 区，若直接在此基础上运行标准 GRPO（奖励为策略命中与否），这 81% 的样本将产生全零奖励，训练等同于在 19% 的 Question 样本上做强化学习，只会进一步强化 Question 偏见。

**3. Agent 方案验证了大模型本身具备多策略知识**

使用零微调的 Qwen2.5-7B-Instruct，仅通过系统提示提供策略定义和少量示例，就在 7 种 SFT 完全失效的策略上获得了 4.7%~40.7% 的命中率，证明模型的预训练知识中已包含足够的情感支持能力，只是被 SFT 的 Question 偏见压制了。

**4. Question 策略的两种学习方式体现了不同的"学习什么"**

SFT 学到的是：看到 ESConv 风格的对话就输出 `[Question]` 标签（机械模式匹配，ci=0.85）。Agent 学到的是：什么情绪状态下提问是合适的，什么时候应该先共情（更接近真实决策能力，但与数据集标注风格不符，故准确率低）。这两种方式的差异本身就说明，"策略准确率"这个评估指标的上限受数据集标注主观性的制约。

## 十一、后续计划

当前最大的挑战是：如何让 GRPO 在 81% UK 的知识分布下有效工作。计划从以下方向推进：

**方向一（首选）：双模式 GRPO**
对 HK/WK 样本用策略命中奖励，对 UK 样本用基于熵的探索奖励（鼓励模型尝试非 Question 策略），避免大量零梯度问题。代码已在 `scripts/train_grpo.py` 中实现。

**方向二：课程学习**
第一阶段只用 HK+WK 样本（19%）训练，等策略分布稳定后再逐步引入 UK 样本。

**方向：Others 策略细化**

将 Others 拆分为 Opening / Closing / Acknowledgment 三个语义明确的子类别，减少标签歧义，让模型有机会真正学会这个占比 17.5% 的类别。

**方向：Agent + GRPO 结合**

以 Agent 的策略推理能力作为 GRPO 的初始化方向，用数据集的策略分布作为奖励信号持续优化，融合两种方法各自的优势。
